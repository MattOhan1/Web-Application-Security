🔐 Common Web Application Security Threats

1. 🧪 Cross-Site Scripting (XSS)

Attackers inject malicious scripts into trusted websites. These scripts run in a user’s browser and can steal cookies, session tokens, or other sensitive data.

2. 🛠️ SQL Injection

A type of attack where hackers insert SQL queries into input fields to gain unauthorized access to data or make changes to the database. Prevent it by:

  • Using prepared statements
  • Validating all user inputs

3. 🎭 Cross-Site Request Forgery (CSRF)

This trick forces users to perform actions they didn't intend while logged in. For example, changing their password or making a transaction without knowing it.

4. 🚫 Broken Authentication

If login systems aren’t secure, attackers can hijack accounts using leaked passwords, session IDs, or brute-force methods.

5. 🔓 Insecure Direct Object References (IDOR)

This flaw allows users to access things they shouldn’t — like another user’s data — by guessing or modifying a URL parameter or ID.

6. 🔍 Security Misconfiguration

Happens when servers, databases, or web apps are set up with default settings, outdated software, or open directories — making them easy targets.
